# LK 4005 拨盘改造说明

这份文档只讨论一种方案：

- 拨盘电机从 `DJI 2006` 改成 [Lk_motor.hpp](F:/H_SG_Gimbal/User/BSP/Motor/LK/Lk_motor.hpp) 的 `LK4005`
- 拨盘输出不使用单电机 `ctrl_Torque()`
- 拨盘输出统一使用 `ctrl_Multi()` 广播模式控制

## 1. 先说结论

你当前 [ShootTask.cpp](F:/H_SG_Gimbal/User/Task/ShootTask.cpp) 的拨盘位置环已经在用累加角：

```cpp
BSP::Motor::Dji::Motor2006.getAddAngleDeg(1)
```

所以换成 LK 4005 后，拨盘业务层大部分逻辑都可以保留：

- `Dail_target_pos -= 40.0f`
- `Dail_target_pos -= angle_per_frame`
- `pid_Dail_pos`
- `pid_Dail_vel`
- `Jamming()`

真正要改的是三件事：

1. 拨盘反馈对象从 `Dji::Motor2006` 换成 `LK::Motor4005`
2. 拨盘发送从 DJI 总线改成 `LK::Motor4005.ctrl_Multi()`
3. LK 电机需要显式 `On()`，并确保 `Parse()` 走对 CAN

## 2. 为什么这次要用 `ctrl_Multi()`

你当前 [Lk_motor.hpp](F:/H_SG_Gimbal/User/BSP/Motor/LK/Lk_motor.hpp) 里已经有：

```cpp
void ctrl_Multi(const int16_t iqControl[4])
```

它的特点是：

- 广播 ID 固定发 `0x280`
- 一帧同时给 4 个 LK 电机下发力矩电流命令
- 每个槽位是一个 `int16_t`
- 当前实现默认走 `CAN1`

所以如果你的拨盘 4005 只是其中一个电机，最小做法不是再给它单独发一帧，而是：

- 给 4 个槽位准备一个长度为 4 的数组
- 拨盘所在槽位填目标力矩
- 其他槽位填 `0`
- 用一次 `ctrl_Multi()` 发出去

## 3. 当前 LK 驱动里你可以直接利用的东西

你现在的 [Lk_motor.hpp](F:/H_SG_Gimbal/User/BSP/Motor/LK/Lk_motor.hpp) 已经有这些关键内容：

- `inline LK4005<4> Motor4005{0x140, {1, 2, 3, 4}, {1, 2, 3, 4}};`
- `ctrl_Multi(const int16_t iqControl[4])`
- `ctrl_Torque(uint8_t id, int16_t torque)`
- `On(uint8_t id)`
- `Off(uint8_t id)`
- `ClearErr(uint8_t id)`

既然你明确要广播模式，那就直接围绕这套现成对象写，不需要再单独实例化单电机版 `Motor4005`。

## 4. 当前拨盘链路

现在拨盘控制链路是：

- 反馈解析：

```cpp
BSP::Motor::Dji::Motor2006.Parse(rx_frame2);
```

- 位置和速度读取：

```cpp
BSP::Motor::Dji::Motor2006.getAddAngleDeg(1);
BSP::Motor::Dji::Motor2006.getVelocityRpm(1);
```

- 拨盘扭矩发送：

```cpp
BSP::Motor::Dji::Motor3508.setCAN(target_Dail_torque, 1);
BSP::Motor::Dji::Motor3508.sendCAN();
```

改成 LK 广播模式后：

- 反馈对象要换成 `BSP::Motor::LK::Motor4005`
- 输出改成 `BSP::Motor::LK::Motor4005.ctrl_Multi(...)`

## 5. 最小改动范围

最小改动只需要关注这两个业务文件：

1. [Task/CallBack.cpp](F:/H_SG_Gimbal/User/Task/CallBack.cpp)
2. [Task/ShootTask.cpp](F:/H_SG_Gimbal/User/Task/ShootTask.cpp)

前提是：

- 你认可 [Lk_motor.hpp](F:/H_SG_Gimbal/User/BSP/Motor/LK/Lk_motor.hpp) 里当前 `Motor4005{0x140, {1,2,3,4}, {1,2,3,4}}` 的 ID 配置
- 你的 LK 4005 的确接在 `CAN1`

如果这两个前提不成立，再回头改驱动配置。

## 6. `CallBack.cpp` 的最小改法

你要把拨盘反馈从 DJI 解析改成 LK 解析。

如果 LK 4005 接在 `CAN1`，就在 [Task/CallBack.cpp](F:/H_SG_Gimbal/User/Task/CallBack.cpp) 的 `HAL_CAN_RxFifo0MsgPendingCallback()` 里加：

```cpp
BSP::Motor::LK::Motor4005.Parse(rx_frame1);
```

如果拨盘仍然接在 `CAN2`，那就不能直接用当前 `ctrl_Multi()`，因为它现在固定发 `CAN1`。这种情况下要先统一硬件或驱动。

## 7. `ShootTask.cpp` 的最小改法

### 7.1 反馈读取改成 LK 4005

把所有拨盘反馈读取从：

```cpp
BSP::Motor::Dji::Motor2006
```

改成：

```cpp
BSP::Motor::LK::Motor4005
```

最少改这些点：

```cpp
float current_angle = BSP::Motor::LK::Motor4005.getAddAngleDeg(1);
auto DailVel = BSP::Motor::LK::Motor4005.getVelocityRpm(1);
auto Dail_pos = BSP::Motor::LK::Motor4005.getAddAngleDeg(1);
```

说明：

- 继续使用 `getAddAngleDeg(1)`
- 不要改回 `getAngleDeg(1)`

### 7.2 给 LK 4005 增加使能

LK 电机要显式使能。

在 `ShootTask()` 启动后，最小可以这样写：

```cpp
static bool dial_enabled = false;
if (!dial_enabled)
{
    BSP::Motor::LK::Motor4005.ClearErr(1);
    BSP::Motor::LK::Motor4005.On(1);
    dial_enabled = true;
}
```

这里默认拨盘电机占用 `Motor4005` 的第 1 个槽位。

如果拨盘实际不是第 1 个槽位，就把这里的 `1` 改成你的实际电机编号。

### 7.3 `CAN_Send()` 改成 `ctrl_Multi()` 广播模式

当前拨盘发送是：

```cpp
BSP::Motor::Dji::Motor3508.setCAN(target_Dail_torque, 1);
```

改成广播模式后，拨盘双环 PID 的最终输出值，直接填到广播数组 `iq_control[4]` 的对应槽位里。

也就是说：

- 位置环 `pid_Dail_pos` 先算出目标速度
- 速度环 `pid_Dail_vel` 再算出最终输出
- 最终这个输出值，就是要塞进 `ctrl_Multi()` 的力矩/电流指令槽位

最小做法是先准备一个 4 元素数组：

```cpp
int16_t iq_control[4] = {0, 0, 0, 0};
```

然后把拨盘所在槽位写入目标扭矩，其余保持 `0`。

如果拨盘电机是 `Motor4005` 的第 1 个槽位：

```cpp
iq_control[0] = static_cast<int16_t>(
    Tools.clamp(target_Dail_torque, 2048.0f, -2048.0f));
```

最后发送：

```cpp
BSP::Motor::LK::Motor4005.ctrl_Multi(iq_control);
```

所以 [Task/ShootTask.cpp](F:/H_SG_Gimbal/User/Task/ShootTask.cpp) 里的 `CAN_Send()` 推荐改成：

```cpp
void Class_ShootFSM::CAN_Send(void)
{
    BSP::Motor::Dji::Motor3508.setCAN(Adrc_Friction_L.getU(), 2);
    BSP::Motor::Dji::Motor3508.setCAN(Adrc_Friction_R.getU(), 3);
    (void)BSP::Motor::Dji::Motor3508.sendCAN();

    int16_t iq_control[4] = {0, 0, 0, 0};
    iq_control[0] = static_cast<int16_t>(
        Tools.clamp(target_Dail_torque, 2048.0f, -2048.0f));
    BSP::Motor::LK::Motor4005.ctrl_Multi(iq_control);
}
```

如果拨盘是第 2 个槽位，就改成：

```cpp
iq_control[1] = ...
```

以此类推。

### 7.4 这和你现在的双环 PID 是怎么对应的

你当前拨盘双环 PID 的控制链本身不用改：

```cpp
pid_Dail_pos.setTarget(Dail_target_pos);
pid_Dail_pos.GetPidPos(Kpid_Dail_pos, Dail_pos, 16384.0f);

pid_Dail_vel.setTarget(pid_Dail_pos.getOut());
pid_Dail_vel.GetPidPos(Kpid_Dail_vel, DailVel, 16384.0f);

target_Dail_torque = pid_Dail_vel.getOut();
```

改成 LK 广播模式后，只改最后一步发送：

```cpp
int16_t iq_control[4] = {0, 0, 0, 0};
iq_control[0] = static_cast<int16_t>(
    Tools.clamp(target_Dail_torque, 2048.0f, -2048.0f));
BSP::Motor::LK::Motor4005.ctrl_Multi(iq_control);
```

也就是说：

- `pid_Dail_vel.getOut()` 仍然是拨盘控制最终输出
- 只是这个输出不再发给 `Motor3508.setCAN(..., 1)`
- 而是写进 `iq_control[]` 对应拨盘槽位，再用 `ctrl_Multi()` 广播发出

## 8. 这次哪些逻辑不用动

下面这些逻辑都可以原样保留：

- `Dail_target_pos -= 40.0f`
- `Dail_target_pos -= angle_per_frame`
- `pid_Dail_pos`
- `pid_Dail_vel`
- `Jamming()`
- `hz_to_angle()`

也就是说，这次不是重写拨盘控制，只是把拨盘电机接口从 DJI 替换成 LK 广播力矩控制。

## 9. 三个关键注意点

### 9.1 `ctrl_Multi()` 的数组顺序就是广播槽位顺序

当前实现里：

```cpp
inline LK4005<4> Motor4005{0x140, {1, 2, 3, 4}, {1, 2, 3, 4}};
```

通常就按 1 到 4 号电机对应：

- `iqControl[0]` -> 1 号
- `iqControl[1]` -> 2 号
- `iqControl[2]` -> 3 号
- `iqControl[3]` -> 4 号

所以你必须先确认：

- 拨盘 4005 在第几个槽位

否则你可能把力矩发到别的 LK 电机上。

### 9.2 力矩命令上限是 `2048`

`ctrl_Multi()` 发的是 `int16_t` 原始电流命令，不是 Nm。

所以最小改动时建议先直接限幅：

```cpp
Tools.clamp(target_Dail_torque, 2048.0f, -2048.0f)
```

但这只是先跑起来，后面大概率还要重调：

- `Kpid_Dail_pos`
- `Kpid_Dail_vel`
- 卡弹判据

因为你当前拨盘 PID 是围绕 DJI 2006 调的。

### 9.3 `ctrl_Multi()` 现在固定走 `CAN1`

当前驱动里广播发送是：

```cpp
frame.id = 0x280;
HAL::CAN::get_can_bus_instance().get_can1().send(frame);
```

所以：

- LK 4005 必须也在 `CAN1`
- `Parse()` 也要在 `CAN1` 回调里处理

如果你的硬件实际接在 `CAN2`，先不要直接按这份文档改。
