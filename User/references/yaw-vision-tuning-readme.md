# 视觉模式 Yaw 调参 README

## 目的

这份文档用于记录当前工程中视觉模式 `yaw` 的控制链路、主要参数含义，以及一套能直接上车使用的调参顺序。

适用范围：

- `Task/GimbalTask.cpp`
- `APP/Variable.cpp`
- `Algorithm/PID.cpp`
- `Algorithm/LADRC/Adrc.cpp`

## 当前控制链路

当前视觉模式下，`yaw` 走的是这条链路：

1. 视觉给出目标 `yaw` 角度，单位为度。
2. `Task/GimbalTask.cpp` 先根据当前 IMU `yaw` 做包角处理。
3. 角度环 `PID` 根据角度误差输出目标角速度。
4. 速度环 `ADRC` 根据目标角速度输出最终力矩。
5. `ctrl_Mit(..., torque)` 将力矩发送给 `DM4310` 的 `yaw` 电机。

核心原则：

- 角度环输出的是目标角速度。
- 速度环输出的才是最终力矩。
- 视觉模式下，角度环不要再直接驱动力矩输出。

## 重点文件

### `Task/GimbalTask.cpp`

这里负责：

- 视觉 `yaw` 的控制主链路
- 角度环输出限幅
- 最终 `yaw` 力矩限幅

### `APP/Variable.cpp`

这里负责：

- `Adrc_yaw_vel` 初始参数
- `Kpid_yaw_angle` 初始参数

### `Algorithm/PID.cpp`

这里要特别注意：

- 当前 `PID` 的 `D` 不是直接对陀螺仪角速度做阻尼
- 它是对误差做跟踪微分后的结果

所以位置环 `D` 不能按常规“速度阻尼”去理解。

### `Algorithm/LADRC/Adrc.cpp`

这里要特别注意：

- `Kp` 是主要响应增益
- `Kd` 乘的是目标 `TD` 的微分量，不是测量速度反馈
- `wc` 是观测器带宽
- `b0` 是对象增益估计

所以这个工程里的 `ADRC Kd` 也不能当作普通速度阻尼参数来硬加。

## 调参前注意事项

- 先调速度环，再调角度环
- 不要同时调 `yaw` 和 `pitch`
- 不要一上来就在真实视觉大步进下调参
- 位置环 `D` 先设为 `0`
- 速度环 `ADRC` 的 `Kd` 先设为 `0` 或极小

原因：

- 当前工程里，位置环 `D` 和 `ADRC Kd` 都会受目标跳变影响
- 如果视觉目标本身有突变，过大的 `D` 或 `Kd` 容易造成额外冲击和超调

## 推荐起始限幅

先用保守限幅：

- 视觉模式目标角速度限幅：`2.0 ~ 3.0 rad/s`
- 视觉模式最终力矩限幅：`2.0 ~ 3.0 Nm`

如果系统稳定但偏慢，再逐步放大。

## 推荐起始参数

### 外环角度 PID

建议起始值：

- `kp = 5.0`
- `ki = 0`
- `kd = 0`

含义：

- 外环主要负责把角度误差变成目标角速度
- 真正的阻尼主要靠内环速度控制，不靠大位置环 `D`

### 内环速度 ADRC

参考当前工程结构，建议起点：

- `TD.r = 80`
- `Kp = 10`
- `Kd = 0`
- `wc = 18`
- `b0 = 20`
- `max = 3.0`

允许调整范围：

- `TD.r = 60 ~ 100`
- `Kp = 8 ~ 12`
- `Kd = 0`
- `wc = 15 ~ 25`
- `b0 = 15 ~ 25`
- `max = 2.5 ~ 3.0`

## 调参顺序

### 第一步：先确认方向

先确认下面三件事：

- 正的目标角速度会不会让 `yaw` 朝正确方向转
- 正的角度误差会不会生成正确方向的目标角速度
- `gyro Z` 的正负方向和控制器正负方向是否一致

如果方向错了，不要继续调参，先修方向。

### 第二步：先单独调速度 ADRC

先不要让视觉直接给大角度跳变。

优先做小速度指令测试：

- `0 -> +0.5 rad/s -> 0`
- `0 -> -0.5 rad/s -> 0`
- `0 -> +1.0 rad/s -> 0`
- `0 -> -1.0 rad/s -> 0`

观察点：

- 跟随是否够快
- 停止是否干净
- 是否有持续抖动
- 正反方向表现是否接近

调整规则：

- 跟随慢：先加 `Kp`
- 抖动、发硬、噪声大：先降 `wc`
- 明显没劲：在 `Kp` 不低的情况下，再小幅减 `b0`
- 一给目标就猛冲：先保持 `Kd = 0`，不要先加 `Kd`

通过标准：

- 小速度指令能稳定跟上
- 停下来不拖尾
- 不持续振荡

### 第三步：再调角度 PID

等速度环稳定以后，再做角度阶跃测试：

- `0 -> 5 deg`
- `0 -> 10 deg`
- `0 -> 20 deg`

观察点：

- 跟随速度
- 接近目标时是否冲过头
- 最终收敛是否干净

调整规则：

- 跟不上：加 `kp`
- 超调大：先降角度环 `kp`
- 只有大步进超调：先降目标角速度限幅
- 小角度发软：小幅提高 `kp`

不要一开始就加较大的 `kd`。

通过标准：

- 小角度响应干净
- 中等角度响应够快
- 大角度不会明显超调

### 第四步：最后回到真实视觉

当手动测试下的内外环都稳定以后，再接回真实视觉目标。

重点检查：

- 视觉目标是否跳变过大
- 视觉更新周期是否稳定
- 丢目标时是否出现目标突跳

如果手动测试稳定、接视觉后变差，优先怀疑视觉链路，不要先怀疑电机参数。

## 常见现象对应处理

### 现象：慢，但很稳

按这个顺序处理：

1. 提高角度环 `kp`
2. 提高视觉目标角速度限幅
3. 提高速度环 `ADRC Kp`

### 现象：大步进超调

按这个顺序处理：

1. 降低视觉目标角速度限幅
2. 降低角度环 `kp`
3. 降低速度环整体攻击性
4. 最后才考虑加一点点位置环 `kd`

### 现象：高频抖动

按这个顺序处理：

1. 降低 `ADRC wc`
2. 降低 `ADRC Kp`
3. 检查 IMU 噪声和机械松动

### 现象：正反方向手感不一样

优先检查：

- 方向定义
- 机械摩擦
- 线束拖拽
- 安装预紧

## 日志建议

调参时建议至少记录这几项：

- 目标 `yaw` 角度
- 当前 `yaw` 角度
- 目标 `yaw` 角速度
- 当前 `gyro Z`
- 最终 `yaw` 力矩输出

如果条件允许，最好把：

- 手动测试输入
- 真实视觉输入

在相同参数下做对比。

## 回退原则

如果某次调参让效果明显变差：

1. 先回退到上一组稳定参数
2. 下一次只改一个参数
3. 用同样的测试动作重新比较

系统不稳定时，不要一次改多个参数。
